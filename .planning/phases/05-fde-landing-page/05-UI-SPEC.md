## Spacing Scale

Declared values (multiples of 4, all already in active use in the codebase — confirmed via `hero.tsx`, `outcomes.tsx`, `final-cta.tsx`, `service-sequence.tsx`, `sticky-cta-bar.tsx`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/inline gaps (rare — icons avoided in this phase) |
| sm | 8px | Compact inline spacing (e.g. baseline gaps in numbered labels) |
| md | 16px | Default page gutter (`px-4`), default content gaps |
| lg | 24px | Section gutter at `sm:` breakpoint (`px-6`), card/step gaps (`gap-6`) |
| xl | 32px | Layout gaps between major elements (`gap-8`) |
| 2xl | 48px | Section internal gaps (`gap-12`), hero flex gap |
| 3xl | 64px | Section vertical padding (`py-16`), hero top padding |

Exceptions (already in active use — confirmed, not new):
- **40px** (`gap-10`) — sequence/flow item gaps in `service-sequence.tsx` / `engagement-flow.tsx`. Keep for the renamed ProcessTransparency section.
- **80px** (`sm:py-20`) — section vertical padding at the `sm:` breakpoint (Outcomes, Hero).
- **96px** (`sm:py-24`) — FinalCta's `sm:` vertical padding (its band deserves the most breathing room as the last conversion moment).
- **44px minimum touch target** (`h-11`, i.e. `h-11 w-full`) — the sticky mobile CTA bar's button and the calculator's number inputs/submit affordance on mobile must not go below this.

## Typography

All values already established in the codebase (`hero.tsx`, `outcomes.tsx`, `service-sequence.tsx`, `final-cta.tsx`, `engagement-flow.tsx`) — reused, not redefined.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`text-base`) | 400 regular, IBM Plex Sans | 1.6 |
| Label | 14px (`text-sm`), uppercase, `tracking-[0.02em]` | 600 semibold, IBM Plex Sans | 1.4 |
| Heading | 20–28px (`text-xl` for sub-headings; `text-2xl`/`1.75rem` for section H2s; explicit `28px` for the emphasized first item in a numbered sequence) | 600 semibold, Fraunces | 1.2 |
| Display | `clamp(32px, 8vw, 44px)` | 600 semibold, Fraunces | 1.1 |

Weight count: exactly 2 (400 regular / 600 semibold) — matches IBM Plex Sans's installed weight subset (`weight: ["400", "600"]`), no other weights available or needed.

**Display is reserved for the Hero H1 only** (the gap statement) — it is the single largest, most visually dominant element on the page, consistent with the existing Hero component's own inline comment establishing it as "the only Display-size text... every section below is visually subordinate." Do not apply Display size anywhere else, including the ROI calculator's result numbers (those use the Heading role, per Color contract below).

### 95%-stat citation (Hero)

Render as a **small (`text-sm`, `text-muted-foreground`) line directly beneath the headline stat/statement** — not a superscript footnote system (disproportionate for a solo-maintained single citation). Required scope-accurate wording per RESEARCH Pitfall 3:

> "95% of **enterprise** generative-AI pilots fail to show measurable ROI (MIT NANDA, 2025)."

Then bridge to SMBs in the following body copy (do not let the stat stand alone as an unscoped universal claim): the same brittle-workflow problem exists in SMBs — worse, because they can't afford the forward-deployed engineering enterprises use to close it.

### Pricing copy (Offer section, LAND-02 / Pitfall 4)

Every price MUST ship paired with scope-qualifying language tied to the free audit — never a bare unconditional number as a headline:

> Free audit (no cost, no commitment) → most engagements: **one-time setup under $10k**, scoped during your free audit → **ongoing retainer under $2k/month**.

Reuse the existing `service-sequence.tsx` numbered/asymmetric layout pattern (step 1 widest, most copy, accent-colored price) — already satisfies the "not a uniform 3-card grid" anti-pattern gate.

### Worst-case/conservative framing (Outcomes, LAND-05)

Outcomes section must include an explicit standalone line carrying this framing — not just implied by conservative numbers:

> "Even in the worst case, you come out ahead."

### ROI Calculator Contract (PROOF-02 / PROOF-03)

Client Component (`"use client"`), zero network calls, zero persistence, zero new dependencies beyond `useState` (matches RESEARCH Pattern 2 exactly — treat the illustrative code in 05-RESEARCH.md as the implementation baseline, not just inspiration).

| Field | Type | Default | Validation |
|-------|------|---------|------------|
| Hours/week lost to manual work | `<input type="number">` | `10` | Clamp non-negative: `Math.max(0, Number(value) || 0)` |
| Team's hourly cost | `<input type="number">` | `35` | Clamp non-negative: `Math.max(0, Number(value) || 0)` |

Derivation (conservative — reinforces LAND-05's worst-case framing inside the calculator, not just the Outcomes section):
- `WEEKS_PER_YEAR = 50` (not 52 — deliberately excludes ~2 weeks holiday/slow periods)
- `annualHours = hoursPerWeek * 50`
- `annualDollars = annualHours * hourlyCost`

Output copy must speak TIME, EFFICIENCY, and PROFIT explicitly (PROOF-03):
- **TIME:** "{annualHours} hours/year back"
- **PROFIT:** "roughly ${annualDollars} in recovered time"
- **EFFICIENCY:** since it has no natural raw number, represent it qualitatively in the same sentence — e.g. "even before counting the efficiency gains from work that no longer needs a person at all"

Empty state: when `hoursPerWeek === 0`, show the Empty State copy from the Copywriting Contract instead of a `$0`/`0 hours` result.

Terminates in a `<BookCta />` render — the calculator's output is not a dead end, it must route back to the audit CTA (PROOF-03's explicit requirement).

### Anonymity scrub checklist (LAND-06 + RESEARCH Pitfall 2 — separate from the literal v1-vocabulary grep)

Run both greps as explicit exit-criterion checks, not a visual pass:
1. `grep -ril "missed call\|intake triage\|dental\|home services\|law firm\|real estate" src/ public/` → must return zero hits
2. `grep -rn "founderName\|Annie\|\bshe\b\|\bher\b" src/` → every hit must be either removed from rendered output or be a code-comment-only reference that never reaches the browser. Known call sites to fix: `cal-embed.tsx` (error fallback), `app/book/page.tsx` (intro copy), `founder-strip.tsx`/`founder-block.tsx` (delete entirely — see Component Inventory above).

`siteConfig.founderName` constant itself may remain in `site.ts` (internal-only, per RESEARCH's Anti-Patterns note) — the requirement is that no *consuming component* interpolates it into visitor-facing copy.
