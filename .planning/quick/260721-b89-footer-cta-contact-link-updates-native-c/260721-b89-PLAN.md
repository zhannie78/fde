## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Visitor browser -> Formspree (external, direct) | Contact form POSTs name/email/message straight from the browser to a third-party (formspree.io), bypassing our server entirely |
| Visitor browser -> `/api/book` (our server) | Untrusted booking input (name/email/note/dateISO) crosses into our Route Handler |
| `/api/book` -> Telegram Bot API (external) | Server-to-server call carrying user-entered text into a Telegram message, using a secret bot token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-b89-01 | Tampering | `/api/book` request body | mitigate | zod schema validates/bounds `name`/`email`/`note`/`dateISO` shape and length at the route entry before any use |
