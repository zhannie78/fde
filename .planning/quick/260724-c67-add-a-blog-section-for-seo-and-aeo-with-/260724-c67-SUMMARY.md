## Decisions Made

- Followed the plan's `content_model_decision` exactly: plain TSX modules, no MDX/CMS/database.
- Blog listing cards use a simpler soft-hover-lift treatment (`border-border` + `hover:-translate-y-0.5` + `hover:shadow-lg` + `hover:border-primary`) rather than the site's `.glow-box` component, since `.glow-box`'s CSS comment explicitly scopes its multi-color animated glow to "stat/offer/process/portfolio cards" — reusing it verbatim for blog list items would have been off-scope reuse, not the intended "soft-hover card language" the plan referenced.
- Removed a speculative `eslint-disable-next-line react/no-danger` comment near the JSON-LD `<script>` after confirming the project's `eslint-config-next` flat config doesn't enable that rule — an unused-directive would itself have been a lint failure.
