<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| visitor URL → /blog/[slug] route | The `slug` path segment is untrusted user input |
| rendered HTML → search/AI crawlers | JSON-LD is machine-parsed by third parties |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-c67-01 | Information Disclosure / Tampering | /blog/[slug]/page.tsx | mitigate | Resolve slug only via `getPostBySlug`; call `notFound()` on miss so arbitrary/probing slugs return 404 rather than rendering. `generateStaticParams` pre-renders only known slugs. |
| T-c67-02 | Tampering (XSS) | JSON-LD script injection | mitigate | Build the JSON-LD object as a JS object and `JSON.stringify` it; all field values come from the trusted in-repo `meta` (no request-derived data), so no untrusted string reaches `dangerouslySetInnerHTML`. |
| T-c67-03 | Tampering (deps) | package installs | accept | No new packages installed — content is plain TSX, no MDX/CMS dependency added. Nothing to audit. |
</threat_model>
