<objective>
Two animation additions to the homepage's "How It Works" (ProcessTransparency) section, following the
existing GSAP/ScrollTrigger scroll-story pattern in scroll-story-provider.tsx (5-act structure, one
`useGSAP` hook, `gsap.matchMedia()` reduced-motion gate):

1. The three steps (Free Audit / Build & Deploy / Retainer) load in sequentially as the section scrolls
   into view, instead of appearing all at once.
2. The "Build & Deploy" step's title animates as a typewriter — letters typed out one at a time — using
   GSAP's TextPlugin (bundled free in the installed gsap@3.15 — all plugins became free as of GSAP 3.13).

Purpose: user request — "have the 3 steps in 'how it works' load sequentially, and have 'Build & Deploy'
be like a typewriter with the letters being typed out."

Design decisions:
- Sequential reveal reuses the exact `ScrollTrigger.batch(..., { onEnter: () => gsap.from(batch, {
  stagger }) })` pattern already used for Act 2 (TheFix cards), targeting the existing
  `[data-process-step]` attribute (already present on each `<li>`, currently unused by any animation).
- Typewriter targets a NEW `data-typewriter` attribute added only to the Build & Deploy step's title
  (via a `typewriter: true` flag on that one entry in the `steps` array) — the other two titles are
  untouched, matching the request's scope (only this one step "is like a typewriter").
- The animated string is read from the DOM (`el.textContent`) at effect-setup time rather than
  hardcoded in scroll-story-provider.tsx, mirroring the existing Act 3a count-up's technique of
  capturing original text before manipulating it — keeps the copy single-sourced in the `steps` array.
- A small `.typewriter-cursor` blinking-caret span sits next to the typed text, colored with the
  section's existing `--lime` "rare accent" token (already used for the 01/02/03 step numbers) rather
  than introducing a new color. It lives inside the same `<li>` that Act 5a's stagger-in fades in, so it
  is invisible until that step has scrolled into view — no extra visibility guard needed.
- Reduced-motion safety: the stagger-in and typewriter tween both live inside the existing
  `no-preference` matchMedia branch only (DSGN-03 — zero ScrollTrigger instances under `reduce`), so
  reduced-motion visitors see the full static "Build & Deploy" text immediately, as today. The cursor's
  CSS `animation` is neutralized by the project's existing global
  `@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important } }`
  rule in globals.css — no new reduced-motion handling needed there either.
</objective>
